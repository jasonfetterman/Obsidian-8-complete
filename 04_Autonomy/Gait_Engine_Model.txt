Gait Engine Model — REV B (High-Torque Quadruped Control)

OVERVIEW
This document defines the gait generation system for OBSIDIAN-8. The gait engine produces coordinated leg motion patterns that maintain stability, minimize power spikes, and operate within the limits of 150KG-class servos.

--------------------------------------------------

SYSTEM OBJECTIVES

- Maintain static and dynamic stability
- Distribute load across multiple legs
- Prevent sudden torque spikes
- Ensure smooth, continuous motion
- Operate safely within actuator limits

--------------------------------------------------

GAIT TYPE

Primary Gait:
- Alternating diagonal gait (trot-style)

Groups:
- Group A: Front Left + Rear Right
- Group B: Front Right + Rear Left

At all times:
- One group supports load
- One group transitions (lift + move)

--------------------------------------------------

GAIT PHASES

Each step cycle consists of two phases:

Phase 0:
- Group A: swing phase (lift and move forward)
- Group B: stance phase (support and stabilize)

Phase 1:
- Group B: swing phase
- Group A: stance phase

Phase Duration:
- Default: 0.5 seconds per phase
- Adjustable based on speed mode

--------------------------------------------------

MOTION COMPONENTS

Each leg movement consists of:

1. Lift Phase
   - Knee joint flexion
   - Reduces ground contact force

2. Swing Phase
   - Hip joint advances leg forward

3. Placement Phase
   - Leg returns to ground smoothly

4. Load Transfer
   - Weight shifts gradually to the newly placed leg

--------------------------------------------------

STABILITY MODEL

Static Stability:
- Minimum of two legs on ground at all times
- Center of gravity must remain within support polygon

Dynamic Stability:
- Controlled via smooth transitions between phases
- Avoid abrupt shifts in load distribution

Constraints:
- No simultaneous lift of all legs
- No rapid center-of-mass shifts

--------------------------------------------------

LOAD MANAGEMENT

Critical Requirement:
- Limit torque spikes on any single joint

Methods:
- Stagger leg lift timing slightly within group
- Smooth motion profiles (handled by motion planner)
- Avoid full load transfer in a single time step

Target:
- Keep individual servo load below 70% of rated torque during normal operation

--------------------------------------------------

SPEED CONTROL

Speed is controlled by:
- Phase duration
- Step length (hip angle offset)

Modes:

1. Slow / Precision Mode
   - Longer phase duration
   - Smaller step size
   - Maximum stability

2. Normal Mode
   - Balanced parameters

3. Fast Mode
   - Shorter phase duration
   - Larger step size
   - Increased instability risk

--------------------------------------------------

ENERGY EFFICIENCY

Goals:
- Minimize unnecessary vertical motion
- Avoid rapid acceleration/deceleration
- Maintain consistent stride timing

Impact:
- Reduced current draw
- Lower thermal load
- Extended runtime

--------------------------------------------------

TERRAIN HANDLING (BASELINE)

Current Capability:
- Flat terrain operation

Limitations:
- No adaptive height control
- No terrain sensing integration

Future Enhancements:
- Foot contact sensing
- Adaptive step height
- Terrain classification integration

--------------------------------------------------

FAILURE CONDITIONS

Instability may occur if:
- Phase timing is too fast
- Load shifts too abruptly
- Servo limits are exceeded

Symptoms:
- Slipping
- Tipping
- Excessive current draw

Mitigation:
- Reduce speed
- Reduce step size
- Increase phase duration

--------------------------------------------------

SYSTEM INTEGRATION

Inputs:
- Desired movement direction and speed
- Base pose from kinematics system

Outputs:
- Target joint angles for all servos

Dependencies:
- motion_scheduler.py (phase timing)
- motion_planner.py (smoothing and limits)
- servo_driver.cpp (execution)

--------------------------------------------------

DESIGN CONSTRAINTS

- Must operate within actuator torque limits
- Must maintain continuous ground contact stability
- Must not generate abrupt motion commands
- Must remain synchronized with control loop timing
