------------------------------------------------------------
1. PURPOSE
------------------------------------------------------------

Define integration rules and validation criteria across all critical domains:

- Structure  
- Propulsion / Actuation  
- Core Systems  
- Electronics  
- Power Distribution  
- Wiring and Harnessing  
- Thermal Management  
- Landing Interface  

Purpose: prevent subsystem conflicts and preserve architecture integrity.  
Reference: /00_System_Definition/V3_Master_System_Definition.txt  

------------------------------------------------------------
2. ARCHITECTURE LOCK
------------------------------------------------------------

The following geometry and structure dimensions are FIXED for OBSIDIAN‑8 V3:

- 45‑inch (1143 mm) footprint  
- 12‑segment 3030 primary ring  
- 8 vertical 3030 posts  
- 2020 secondary bracing system  
- 300 mm central clearance cylinder  
- Carbon‑fiber nylon printed interface parts  
- No primary load path through printed components  

Any modification requires documented revision and updated load validation.  
Reference: /01_Mechanical/Frame_Architecture.txt  

------------------------------------------------------------
3. INTEGRATION HIERARCHY
------------------------------------------------------------

Load Order / Priority:

1. Aluminum structural members  
2. Fasteners (Grade 10.9 minimum)  
3. Secondary aluminum bracing  
4. Printed interface parts (for shear transfer only)  
5. Electronics assemblies  
6. Wiring and harness routes  
7. Thermal control components  

Lower‑priority systems must not compromise higher‑priority structural elements.  

------------------------------------------------------------
4. CLEARANCE MANAGEMENT
------------------------------------------------------------

Reserved volumes:

- Central 300 mm diameter cylinder (core services)  
- 50 mm inward buffer from primary ring (ID protection zone)  
- Upper airflow path (plenum)  

No subsystem infringement in reserved zones without approved Change Request.  
Reference: /01_Mechanical/Clearance_Map.step  

------------------------------------------------------------
5. MASS DISTRIBUTION CONTROL
------------------------------------------------------------

Target All‑Up Mass (AUM): 40 – 55 kg  
Structural Maximum: ≤ 60 kg  

Center of Gravity (Cg):  
- Horizontal offset ≤ 50 mm from geometric center.  
- Vertical position = 35 – 55 % of frame height.  

Subsystem placement must comply with /00_System_Definition/Mass_and_Center_of_Gravity_Model.txt.  

------------------------------------------------------------
6. INTERFACE CONTROL RULES
------------------------------------------------------------

Propulsion Mounts → direct aluminum contact (no printed spacers in load path).  
Electronics Deck → mount to secondary structure only.  
Battery System → must reside within lower core region below mid‑height.  
Wiring → follow route definitions in /02_Electrical/Wiring_Harness_Specification.txt.  
Thermal Management → no sealed heat traps; provide active airflow or passive sink paths.  
Duet 3 and Maestro Boards → mount isolated from vibration zones, cooled via shared heatsink duct.  

------------------------------------------------------------
7. REVISION CONTROL
------------------------------------------------------------

Any geometry change requires:  
1. Updated geometry file (.step / .stl)  
2. Updated cut list (if affected)  
3. Updated mass assumptions (if component weight changes)  
4. Version increment and Change Request approval  

No silent geometry or interface drift permitted.  
All source CAD and manifests stored under /12_STL_Production/00_Master_List/.  

------------------------------------------------------------
8. VALIDATION FLOW
------------------------------------------------------------

Required sequence before integration closure:

1. Structural assembly verification  
2. Torsional rigidity test  
3. Landing load test  
4. Wiring and continuity validation  
5. Thermal performance run  
6. Cg measurement and validation (check Section 5)  
7. Dynamic vibration observation  

Each stage must pass before advancing to the next test phase.  
All results logged in /09_Testing/Integration_Validation_Report.txt.  

------------------------------------------------------------
9. FAILURE ESCALATION RULES
------------------------------------------------------------

If subsystem conflicts are identified, resolve by priority:

1. Structural integrity  
2. Load path accuracy  
3. Cg stability  
4. Thermal safety  
5. Electrical isolation and integrity  
6. Serviceability and accessibility  

Aesthetic or cosmetic considerations have lowest priority.  

------------------------------------------------------------
10. END‑STATE DEFINITION
------------------------------------------------------------

System considered fully integrated when:  
- All geometry files locked and baseline tag applied.  
- Assembly matches design specifications.  
- Structural and load tests passed.  
- Thermal limits respected.  
- Cg validated and logged.  
- No visible structural drift or deviation.  

------------------------------------------------------------
11. DESIGN INTENT
------------------------------------------------------------

This document forms the master reference for end‑to‑end integration of OBSIDIAN‑8 V3.  
Following its rules maintains architecture consistency and ensures mechanical, electrical, and thermal domains interlock safely.  
Any exceptions must be approved through the Configuration Management Plan and logged as a formal change record.
