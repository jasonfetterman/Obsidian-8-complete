Failure Modes and Effects Analysis (FMEA) — REV B (150KG Servo System)

OVERVIEW
This document identifies potential failure modes within OBSIDIAN-8, evaluates their impact, and defines mitigation strategies. This revision reflects the integration of high-torque servo systems, distributed control, and high-current power architecture.

--------------------------------------------------

RISK CLASSIFICATION

Severity Levels:
- LOW: Minor performance degradation
- MEDIUM: Loss of function, recoverable
- HIGH: System instability or shutdown
- CRITICAL: Hardware damage or safety risk

--------------------------------------------------

FAILURE MODE 1 — SERVO OVERCURRENT

Cause:
- Sudden position changes
- Mechanical binding
- Excessive load

Effect:
- Voltage drop
- System instability
- Potential servo damage

Severity:
- HIGH

Mitigation:
- Motion smoothing (motion_planner.py)
- Current-aware gait design
- Power system sized for peak load
- Fuse protection per branch

--------------------------------------------------

FAILURE MODE 2 — POWER BROWNOUT

Cause:
- High simultaneous servo load
- Undersized battery or wiring
- Voltage sag under load

Effect:
- Raspberry Pi reset
- Loss of control
- Unpredictable behavior

Severity:
- CRITICAL

Mitigation:
- Separate logic and servo power rails
- Bulk capacitance on servo lines
- High-discharge battery (50C+)
- Dedicated voltage regulation for logic

--------------------------------------------------

FAILURE MODE 3 — LOSS OF CONTROL SIGNAL

Cause:
- Communication failure (Pi ↔ Teensy)
- Software crash
- Cable disconnection

Effect:
- Frozen or drifting servo positions
- Potential instability

Severity:
- HIGH

Mitigation:
- Teensy watchdog timeout
- Hold-last-position behavior
- Hardware kill line system

--------------------------------------------------

FAILURE MODE 4 — RUNAWAY ACTUATION

Cause:
- Software bug
- Invalid command data
- Memory corruption

Effect:
- Uncontrolled servo movement
- Mechanical damage
- Safety hazard

Severity:
- CRITICAL

Mitigation:
- Command validation
- Motion limits in firmware
- Emergency kill line (hardware cutoff)
- Rate limiting and acceleration limits

--------------------------------------------------

FAILURE MODE 5 — STRUCTURAL FAILURE

Cause:
- Frame not rated for servo torque
- Material fatigue
- Poor fastener integrity

Effect:
- Frame deformation
- Misalignment
- Catastrophic mechanical failure

Severity:
- CRITICAL

Mitigation:
- Reinforced frame design
- Proper material selection (aluminum/steel)
- Regular inspection and torque checks
- Load distribution through structure

--------------------------------------------------

FAILURE MODE 6 — THERMAL OVERLOAD

Cause:
- Sustained high current draw
- Poor ventilation
- Undersized components

Effect:
- Servo overheating
- Battery degradation
- Component failure

Severity:
- HIGH

Mitigation:
- Thermal-aware operation
- Adequate airflow
- Temperature monitoring (thermal_monitor.cpp)
- Conservative duty cycles

--------------------------------------------------

FAILURE MODE 7 — WIRING FAILURE

Cause:
- Poor routing
- Lack of strain relief
- Vibration fatigue

Effect:
- Intermittent signals
- Power loss
- System instability

Severity:
- HIGH

Mitigation:
- Proper cable routing and separation
- Strain relief at all connection points
- Regular inspection

--------------------------------------------------

FAILURE MODE 8 — GROUNDING / NOISE ISSUES

Cause:
- Improper grounding
- Signal interference from high-current lines

Effect:
- Erratic servo behavior
- Communication errors

Severity:
- MEDIUM

Mitigation:
- Single-point ground design
- Separation of power and signal wiring
- Shielding or twisted pair for signals

--------------------------------------------------

FAILURE MODE 9 — GAIT INSTABILITY

Cause:
- Poor phase timing
- Excessive speed
- Uneven load distribution

Effect:
- Slipping
- Tipping
- Increased power draw

Severity:
- HIGH

Mitigation:
- Controlled gait scheduling
- Conservative default parameters
- Gradual speed scaling

--------------------------------------------------

FAILURE MODE 10 — CONNECTOR FAILURE

Cause:
- Underrated connectors
- Vibration loosening
- Heat buildup

Effect:
- Power interruption
- Arcing or damage

Severity:
- HIGH

Mitigation:
- Use XT60/XT90 connectors
- Secure all connections
- Inspect regularly

--------------------------------------------------

SYSTEM-LEVEL SAFETY STRATEGY

Layered Protection:

1. Software Layer
   - Motion limits
   - Watchdog timers

2. Electrical Layer
   - Fuses
   - Voltage regulation
   - Power isolation

3. Hardware Layer
   - Kill switch
   - Structural integrity

No single failure should result in uncontrolled behavior.

--------------------------------------------------

VALIDATION REQUIREMENTS

- Simulated fault injection (software)
- Physical kill switch testing
- Load testing under worst-case conditions
- Thermal stress testing
